Installation d'AbiWord sur Linux / Intel

Bienvenue dans la communauté des utilisateurs d'AbiWord!

La dernière version d'AbiWord doit correctement s'installer et fonctionner sur les systèmes Linux avec processeur Intel compatible. AbiWord a été "packagé" dans plusieurs formats de distribution afin de simplifier son installation sur votre système. Il existe également une implémentation spécifique d'AbiWord pour GNOME afin de mieux y être intégré.

  • DEB -- Debian 2.1 (Slink) ou ultérieur
  • RPM -- RedHat 5.2 ou ultérieur, SuSE 6.1 ou ultérieur
  • SLP -- Stampede Linux
  • TAR -- pour tous

Une version plus exhaustive du processus d'installation est disponible dans le fichier README.

RPM

Le processus d'installation RPM est très simple.

  1. Télécharger l'archive (abisuite-x.x.x-1.i386.rpm).
  2. En tant que superviseur, taper la commande:
      rpm -Uvh abisuite-x.x.x-1.i386.rpm
  3. Si vous utilisez KDE, effectuez un clic droit sur le fichier rpm, et sélectionner kpackage du menu contextuel.

Quand la fenêtre kpackage apparaît, cliquez sur le bouton Install. Si vous avez des librairies partagées non installées, vous aurez à les installer avant de pouvoir installer AbiWord.

  1. Cela installera AbiWord et ses fichiers dans le dossier /usr/share/abisuite. Des liens par défauts appelés AbiWord et abiword seront stockés dans /usr/bin/X11 et pointeront sur les exécutables. L'option -Uvh à la place de l'option -i aura pour effet de remplacer les versions précédentes d'AbiWord pour ceux qui veulent installer une mise à jour.
  2. Pour des informations complémentaires sur les packages RPM, veuillez consulter le fichier README.

DEB

Le processus d'installation DEB est très simple.

  1. Télécharger l'archive (abisuite_x.x.x-1_i386.deb).
  2. En tant que superviseur, taper la commande:
      dpkg -i abisuite_x.x.x-1_i386.deb
  3. Cela installera AbiWord et ses fichiers dans le dossier /usr/share/abisuite. Des liens par défauts appelés AbiWord et abiword seront stockés dans /usr/bin/X11 et pointeront sur les exécutables.
  4. Pour des informations complémentaires sur les packages DEB, veuillez consulter le fichier README.

SLP

Le processus d'installation SLP doit être aussi simple que les processus d'installation DEB ou RPM. Nous ne disposons pas pour le moment de distribution Stampede Linux installée et nous n'avons donc pas testé nos packages Stampede Linux. Ces fichiers ont été automatiquement générés à l'aide d'Alien.

Si vous essayez la distribution SLP, nous vous encourageons à nous envoyer un message dans la liste de développement d'AbiWord faisant part de votre expérience.

  1. Télécharger l'archive (abisuite-x.x.x.slp).
  2. Installer AbiWord via le manager de package Stampede.
  3. Cela installera AbiWord et ses fichiers dans le dossier /usr/share/abisuite. Des liens par défauts appelés AbiWord et abiword seront stockés dans /usr/bin/X11 et pointeront sur les exécutables.
  4. Pour des informations complémentaires sur les packages SLP, veuillez consulter le fichier README.

TAR

Il existe deux versions des archives Tar: une version liens-statiques et une version liens-dynamiques. La version dynamique est moins grosse (2.6Mb contre 3.3Mb), mais utilise un nombre considérable de librairies partagées de votre système. Si vous ne saisissez pas la différence, choisissez la version statique

Le processus d'installation de l'archive statique Tar parait complexe mais est actuellement assez simple.

  1. Télécharger l'archive (abisuite-x.x.x-Linux_i386_static.tar.gz).
  2. Copiez l'archive dans un dossier accessible par vous et ouvrez l'archive: 
      gunzip -c abisuite-x.x.x-Linux_i386_static.tar.gz | tar xvf -

    ... ou, si votre commande tar accepte les tags 'z': 

      tar xvzf abisuite-x.x.x-Linux_i386_static.tar.gz

    Les deux commandes créeront un dossier abisuite-x.x.x-Linux_i386_static.

  3. Allez dans ce dossier en tant que superviseur.
  4. Lancez le script d'installation: 
      ./install.sh
  5. Par défaut, Cela installera AbiWord dans le dossier /usr/local/AbiSuite de votre disque dur. Vous pouvez changer le dossier par défaut si souhaité.
  6. Vous pourrez également créer les liens vers les exécutables dans un dossier bin . Par défaut ce lien est /usr/local/bin.
  7. Merci de consulter le fichier README pour des informations complémentaires.

Le processus d'installation de l'archive dynamique (abisuite-x.x.x-Linux_i386_dynamic.tar.gz) est identique. Remplacer simplement _dynamic par _static dans les étapes 1 et 2 ci-dessus. Si vous voulez plus d'informations à propos des librairies et des versions que la version dynamique utilise, veuillez consulter le fichier README.

L'enregistrement sous Linux

Si vous enregistrer vos documents sous Linux, il est recommandé de les placer dans le dossier /home, à moins que vous n'utilisiez le "dual boot" (vois ci-dessous). Si vous avez un nombre considérable de documents vous devriez les placer par exemple dans le dossier /home/documents. 

Pour enregistrer un document, souvenez vous que vous devez avoir la permission de l'écrire. Parfois, vous pouvez ne pas pouvoir enregistrer un document simplement parce que vous n'en avez pas la permission. Dans ce cas, il est possible de l'"Enregistrer sous..." afin de l'enregistrer dans un autre dossier.

Linux et Windows en "Dual Boot"

Si vous avez à la fois Windows et Linux installé sur deux partitions différentes de votre disque dur, et que vous vouliez utiliser vos documents AbiWord sur les deux systèmes d'exploitation, vous aurez à enregistrer vos documents sur une partition FAT ou FAT32, ou alors sur une disquette. Cela parce que Windows ne peut avoir accès aux partitions Linux . Là encore il est préférable d'enregistrer vos documents sous le dossier par défaut de Windows: "Mes Documents". Souvenez vous toujours que sous Linux, vous devez avoir la permission d'écrire dans ce dossier.

BeOS et Linux en "Dual Boot"

Si vous avez à la fois Beos et Linux installés sur deux partitions différentes de votre disque dur, il est ici préférable d'enregistrer vos documents dans le système de fichier natif propre à BeOS.

KDE and AbiWord

Après avoir installé AbiWord sur votre machine, vous pouvez correctement configurer AbiWord de manière à ce qu'il soit optimisé pour KDE. Pour cela, effectuer un clic droit sur le bureau de KDE et sélectionner Nouveau du menu contextuel, puis Application.

Taper maintenant "AbiWord" dans la fenêtre comme celle ci-dessous.

Ensuite, taper "abiword" dans la fenêtre "Execute" en respectant la casse (tout en minuscule).
Si vous voulez un icône AbiWord sur votre bureau, copiez-le du dossier /usr/share/Abisuite/icons vers le dossier /usr/share/icons. Vous devriez obtenir une fenêtre semblable à celle ci-dessous.

Pour terminer, cliquer sur le bouton OK : l'icône d'AbiWord apparaît alors sur le bureau de KDE. Vous pouvez également le placer dans la barre des tâches simplement en y effectuant un "Drag & Drop".
